Credit Score Anaylsis Project

Overview

In this project you'll analyze data regarding borrower behavior to estimate credit scores. You'll use 27 properties of borrowers to predict a credit score of Good, Standard, or Poor.

The Data

Files train.csv and test.csv are provided. Use the former for training and validation and the latter only for testing. Your predictor must not use any data in test.csv for training or validation.

Expected Data Cleaning

In addition to any other data cleaning, you're expected to do the following:

Wild Outliers and Multiple Records Per Person

There are clearly some wild outliers in several columns, almost certainly due to entry errors. However, fortunately, there are also multiple records per person, so in many cases we can fix the missing value by getting the correct value from other records for the same person.

Correct outliers where feasible with a transformer that uses this fact.
The transformer must be constructed with a hi/lo valid range outside of which it will assume a datum is invalid, and must substitute the mode of the valid data for the person in question. Use this same transformer to fix missing data e.g. in Occupation. Do not use it for data that varies widely even for the same person from record to record.

Loan Type

Write a transformer that expands this to a multicategory binary value, with a 1 for each loan type that applies.

Payment Behavior

Write a transformer that translates this string-value into two numerical values, one for charge sizes and the other for payment sizes.

Rework Credit_History_Age

This verbal description clearly needs to be converted to an integer number of months. Write a transformer to do so.

Make Columns Uniform in Scale

This is optional for any decision tree algorithm, but required for MLP prediction.

Do Any Other Needed Adjustments

The list above covers most of the ground, but there will be some other data cleaning needed. Use your judgment on this.

One big pipeline

In the spirit of the text example e.g. on p. 87, assemble all your data adjustments into a pipeline

Target Test Scores

Once your data is cleaned, try at least two methods for training an estimator, using Scikit-learn tools. One must be a decision tree of some sort, with ensemble improvements. The other may be an MLP or other method of your choice.

You must achieve at least 93% accuracy with test.csv after training. Training may take at most 10 minutes, since you'll be asked to run it during demo. Further improvements past this will result in progressively higher grades.

Submitting

Demonstrate your code to me, and be ready to discuss how you cleaned the data and what ID methods you used. Once it passes style and demos sufficiently accurate test performance, you can submit it.
